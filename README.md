# RocketSim

RocketSim is a Python-based model rocket simulation project with three main workflows:
- A physics simulation engine (6-DOF style state model + PID fin control)
- A desktop GUI for configuring and running flights
- A 3D playback visualizer for the generated flight data

This project was built as an exam project and has grown into a practical sandbox for rocket stability/recovery analysis.

## What It Can Do

- Simulate a rocket flight from launch to landing
- Load real motor thrust curve CSV data
- Model:
  - Thrust over time (interpolated from motor file)
  - Gravity and altitude-based air density
  - Aerodynamic drag
  - Fuel mass burn-down
  - Wind drift (altitude-scaled)
  - Rail-constrained launch behavior
  - Parachute deployment after configurable delay from apogee
- Run a simple SITL-style PID control loop for fin actuation
- Compute and report:
  - Apogee, max velocity, max Mach
  - Impact velocity
  - Recommended field size (based on wind and descent time)
  - Required parachute size for target landing speed
  - Opening shock load and suggested harness strength
  - Static stability margin (calibers)
- Save outputs to CSV/JSON for analysis and visualization
- Visualize:
  - 2D telemetry plots in the GUI
  - 3D rocket trajectory replay in Ursina

## Project Files

- `main.py`: simulation engine, report generation, CSV export
- `main_gui.py`: PyQt desktop app (builder + flight config + telemetry)
- `visualizer.py`: Ursina-based 3D playback viewer (reads `flight_data.csv`)
- `requirements.txt`: Python dependencies
- `rocket_config.json`: runtime configuration used by `main.py`
- `flight_data.csv`: time-series output generated by simulation
- `flight_report.json`: summary and safety analysis generated by simulation
- `AeroTech_F40W.csv`: included motor thrust curve sample
- `rocket.obj`, `Launch_Rail.obj`: 3D assets used by the visualizer

## Requirements

- Python 3.10+
- pip

Install dependencies:

```bash
python -m pip install -r requirements.txt
```

## Quick Start

### 1) Run the GUI (recommended)

```bash
python main_gui.py
```

In the GUI:
- Set vehicle/environment/recovery values
- Ensure `Motor File` points to a valid thrust curve CSV (default: `AeroTech_F40W.csv`)
- Click `RUN SIMULATION`
- Use the chart toggles to inspect altitude/velocity/acceleration
- Click `LAUNCH 3D VISUALIZER` to replay the generated flight

### 2) Run simulation from CLI

Use default config file:

```bash
python main.py
```

Use a specific config file:

```bash
python main.py rocket_config.json
```

### 3) Run 3D visualizer directly

```bash
python visualizer.py
```

The visualizer expects `flight_data.csv` to already exist.

## GUI Overview

### Vehicle Builder tab
- Add/edit/remove modular components (`body`, `nose`, `fins`, `motor`)
- Draw and inspect a simple rocket schematic
- Save/load full projects as `.rkt`

### Flight Telemetry tab
- Configure flight parameters
- Run simulation from inside the app
- Plot selected telemetry curves
- See summary report values and advice pulled from `flight_report.json`

## Visualizer Controls

In the 3D viewer (`visualizer.py`):
- `Space`: arm/start countdown and launch replay
- `R`: reset replay
- Mouse + editor camera controls: orbit/pan/zoom scene

## Input Configuration

`main.py` reads a JSON config with keys like:
- `Dry Mass (kg)`
- `Propellant Mass (kg)`
- `Diameter (m)`
- `CP Dist (m)`
- `CG Dist (m)`
- `Fin Area (m²)`
- `Motor File`
- `Rail Length (m)`
- `Wind Speed (m/s)`
- `Chute Dia (m)`
- `Chute Cd`
- `Chute Delay (s)`

The GUI writes this automatically to `rocket_config.json` before running `main.py`.

## Motor File Format

Motor files are loaded with:
- `pandas.read_csv(..., skiprows=4)`
- Expected columns:
  - `Time (s)`
  - `Thrust (N)`

Use `AeroTech_F40W.csv` as the reference format.

## Outputs

After a simulation run:

1. `flight_data.csv`
Time-series telemetry including:
- Time
- Altitude
- Position (x/z)
- Velocity magnitude
- Acceleration
- Pitch/yaw
- Thrust
- Fin commands
- Recovery deployed flag

2. `flight_report.json`
Summary metrics and safety/recovery/stability analysis.

3. Console report
Printed flight safety and recovery analysis from `main.py`.

## Physics/Model Notes

- Integrator: RK4 for state propagation
- Drag model: quadratic drag
- Atmosphere: simplified ISA-style density using altitude
- Wind: power-law style scaling with altitude
- Recovery: chute drag model after delayed deployment
- Stability metric: `(CP - CG) / diameter` (calibers)

## Known Limitations

- The current control model is simplified and intended for simulation/testing, not flight certification
- The visualizer is a playback tool of recorded data, not a live physics engine
- Some GUI fields are currently informational and not fully used by the simulation core

## Troubleshooting

- `FileNotFoundError` for motor CSV:
  - Verify `Motor File` path in GUI/config points to an existing CSV
- Visualizer opens but no motion:
  - Run a simulation first so `flight_data.csv` is refreshed
- `AttributeError: module 'numpy' has no attribute 'trapezoid'`:
  - Upgrade NumPy (`python -m pip install --upgrade numpy`)
- 3D asset/model issues:
  - Ensure `rocket.obj` and `Launch_Rail.obj` are present in project root
